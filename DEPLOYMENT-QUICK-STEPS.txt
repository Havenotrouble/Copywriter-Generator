════════════════════════════════════════════════════════════════
    Copywriter Generator - 服务器部署快速步骤
════════════════════════════════════════════════════════════════

第一步：推送代码到 Git 仓库
─────────────────────────────
在 Windows 本地执行：

    cd D:/liziyi/codeprivate/Copywriter-Generator
    git remote add origin https://github.com/your-username/Copywriter-Generator.git
    git push -u origin master


第二步：连接到 Linux 服务器
─────────────────────────────
    ssh your-username@your-server-ip


第三步：安装 Docker（首次部署）
─────────────────────────────
Ubuntu/Debian:

    sudo apt update
    sudo apt install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
    
    # 或使用便捷脚本
    curl -fsSL https://get.docker.com -o get-docker.sh
    sudo sh get-docker.sh
    
    # 启动 Docker
    sudo systemctl start docker
    sudo systemctl enable docker


第四步：克隆并配置项目
─────────────────────────────
    # 克隆项目
    git clone https://github.com/your-username/Copywriter-Generator.git
    cd Copywriter-Generator
    
    # 配置环境变量
    cp .env.example .env
    nano .env
    # 填入: DEEPSEEK_API_KEY=sk-your-actual-key


第五步：一键部署
─────────────────────────────
    chmod +x *.sh
    ./deploy.sh


第六步：验证部署
─────────────────────────────
    # 检查服务状态
    ./status.sh
    
    # 健康检查
    ./health-check.sh
    
    # 浏览器访问
    http://your-server-ip


常用命令
─────────────────────────────
    ./deploy.sh          # 部署/更新
    ./status.sh          # 查看状态
    ./logs.sh            # 查看日志
    ./stop.sh            # 停止服务
    ./quick-restart.sh   # 快速重启
    ./health-check.sh    # 健康检查
    ./backup.sh          # 备份

    # 或使用 make
    make deploy
    make status
    make logs
    make health


更新部署
─────────────────────────────
    git pull
    ./deploy.sh


配置防火墙（如需要）
─────────────────────────────
    # Ubuntu/Debian
    sudo ufw allow 80/tcp
    sudo ufw enable
    
    # CentOS/RHEL
    sudo firewall-cmd --permanent --add-service=http
    sudo firewall-cmd --reload


故障排查
─────────────────────────────
    # 查看日志
    docker compose logs backend -f
    docker compose logs frontend -f
    docker compose logs nginx -f
    
    # 重新构建
    docker compose build --no-cache
    docker compose up -d
    
    # 清理空间
    docker system prune -a -f
